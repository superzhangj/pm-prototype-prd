# 需求规格说明书 (PRD) 模板

## 模板说明

本模板综合腾讯、阿里巴巴、京东等大厂的产品需求文档规范，结合 pm-prototype-prd skill 的原型注释系统，实现"原型→PRD 自动映射"。

### 使用场景

| 触发词 | 行为 |
|--------|------|
| "输出需求文档/PRD/需求规格说明书" | 基于当前原型 + 注释，生成完整 PRD |
| "导出需求文档" | 同上，输出 .md 文件 |
| "写个产品需求文档" | 按完整模式输出 |
| "简要版需求文档" | 按精简模式输出（仅功能清单 + 核心注释） |

### PRD 生成规则

1. **注释映射**：原型中的每条注释自动映射为 PRD 中对应的功能需求条目
2. **类型分类**：按注释类型（interaction/business/edgecase/permission）归类到文档不同章节
3. **版本对齐**：PRD 版本号与原型版本标注栏一致
4. **截图嵌入**：将原型关键页面截图（通过框选工具或描述）嵌入文档对应章节
5. **数据支撑**：从原型中的模拟数据推断数据字典、字段约束、校验规则

## 各注释类型到 PRD 的映射规则

以下规则决定了原型中的每条注释如何映射到 PRD 中的对应章节：

| 注释类型 | 映射到 PRD 章节 | 映射产物 |
|---------|----------------|---------|
| `interaction` | 5.功能需求详述（主要） + 6.页面交互说明 | 功能条目 + 操作流程 + 交互规格表 |
| `business` | 4.业务流程 + 5.功能需求详述 | 业务规则 + 状态流转 + 数据逻辑 |
| `edgecase` | 5.功能需求详述（边界与异常） + 9.边界与异常处理 | 异常列表 + 处理策略 |
| `permission` | 3.用户角色与权限 | 权限矩阵条目 + 角色定义 |
| `note` | 10.附录 / 7.数据字典 | 备注说明 / 待确认事项 |

## PRD 生成执行流程

当用户要求输出需求文档时，AI 按以下步骤执行：

1. **读取原型文件**：找到用户指定的 HTML 原型文件，或用 Glob 搜索最近生成的 .html 原型
2. **提取注释数据**：从 HTML 中扫描所有 `__addAnnotationOn` 和 `__addAnnotation` 调用，提取每个注释的：
   - id（编号）
   - title（标题）
   - description（描述）
   - type（类型：interaction/business/edgecase/permission/note）
   - targetSelector（关联的页面元素）
3. **分类整理**：按 type 将注释分组，判断每个功能属于哪个模块
4. **补充需求分析**：结合 Step 1 中的需求分析结果（用户角色、场景、页面结构等）
5. **输出文档**：按上述模板填充，生成完整的 .md 格式 PRD 文件
6. **交付**：保存文件并通过 `deliver_attachments` 工具交付给用户
